- **实施控制阶段**
  
      开发，测试，运维部署

##实现的需求
####乘客端
    1. 发送验证码
        三档验证，防止恶意短信轰炸
    2. 登录/注册
    3. 查看开通区域  --> 高德围栏
    4. 预估价格
    5. 下单
    6. (司机流程）
    7. 支付（分布式事务 订单，支付，积分）
    8. 评价
####司机端
    1. 发送验证码
    2. 登录/注册
    3. 查看/改变司机状态
    4. 司机抢单（分布式锁）
    5. 订单状态变更
    6. 发起收款
####Boss端
    运营
    派单规则配置
    后台管理

##微服务设计原则
   架构，模式，拆分，隔离 -> 目标：隔离系统的变化点 

**具体原则**

    1. 高内聚，低耦合
    2. 高度自治：开发，测试，构建，部署，运行，发布,无状态
    3. 以业务为中心
    4. 弹性设计（容错，降级，隔离）
    5. 自动化 持续集成，持续交付
    6. 粒度把控
**AKF原则**

    X：水平扩展 -> 集群
    Y：业务拆分
    Z：数据分区
